# Unified Deep Dive Layout V2 - Simplified A/B/C Tiers

## Design Principles
- **Simple tiers**: A, B, C (no Hero/Solid/Underperformer labels)
- **Grouped sections**: Separate sections for A, B, C, New, Lost
- **Revenue-sorted**: Within each section, sorted by revenue DESC (highest impact first)
- **Clear transitions**: Show warnings when approaching tier boundaries
- **New/Lost grouping**: Show which tier new items are in, which tier lost items were in

## Key Differences from V1:

### ✅ What Changed:

1. **Simple Tiers**: A, B, C only (no Hero/Solid/Underperformer labels)

2. **Clear Sections**:
   - Each tier is a separate table
   - Sorted by revenue DESC within each tier (highest impact first)

3. **Cumulative % Column**: Shows exactly where item sits in revenue distribution

4. **Transition Warnings**:
   - "⚠️ At threshold" - exactly at 80% or 95%
   - "⚠️ Chuẩn bị xuống B" - in Tier A but cumul > 80%
   - "📈 Gần lên Tier A" - in Tier B but close to 80%

5. **New Section has Group column**:
   - NEW-A, NEW-B, NEW-C (which tier they'd be in)

6. **Lost Section has Group column**:
   - LOST-A, LOST-B, LOST-C (which tier they WERE in)

7. **Impact-based sorting**: Always revenue DESC = highest business impact first

### 🎯 Benefits:

- ✅ Easy comparison within each tier
- ✅ Clear threshold warnings
- ✅ Simple A/B/C mental model
- ✅ New/Lost context preserved (which tier)
- ✅ Business impact prioritized (revenue sort)
