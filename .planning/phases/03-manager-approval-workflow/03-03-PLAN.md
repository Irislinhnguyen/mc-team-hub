  <action>
    Create POST endpoint at apps/web/app/api/challenges/submissions/[id]/approve/route.ts:

    1. **Import dependencies:**
       - import { NextRequest, NextResponse } from 'next/server'
       - import { getServerUser, isAdminOrManager } from '@query-stream-ai/auth/server'
       - import { createAdminClient } from '@query-stream-ai/db/admin'
       - import { notifyLeadersGradeApproved } from '@/lib/services/workflowNotificationService'

    2. **Extract submission ID from URL:**
       - const params = await context.params
       - const submissionId = params.id

    3. **Authentication and authorization:**
       - const user = await getServerUser()
       - if (!user || !isAdminOrManager(user)) return NextResponse.json({ error: 'Forbidden' }, { status: 403 })

    4. **Validate current submission state:**
       - Query submission: select id, status, challenge_id, user_id (student), created_at from challenge_submissions where id = submissionId
       - if (!submission) return 404 Not Found
       - if (submission.status !== 'pending_review') return 400 with error 'Invalid state transition: submission not pending review'

    5. **Find the Leader who submitted for review:**
       - Query approvals: select user_id from approvals where submission_id = submissionId and action = 'submitted_for_review' order by created_at desc limit 1
       - This identifies which Leader to notify

    6. **Update submission status to approved:**
       - update challenge_submissions set status = 'approved' where id = submissionId and status = 'pending_review'
       - if no rows updated, return 400 (state changed concurrently)

    7. **Create approval audit record:**
       - insert into approvals (submission_id, user_id, user_role, action, from_status, to_status)
       - values: (submissionId, user.sub, user.role, 'approved', 'pending_review', 'approved')

    8. **Query challenge for notification:**
       - select id, name from challenges where id = submission.challenge_id

    9. **Trigger notification to Leader:**
       - await notifyLeadersGradeApproved(challenge.id, challenge.name, true, undefined)
       - Note: Only notify the specific Leader who submitted, not all Leaders
       - Wrap in try-catch, log errors but don't fail request

    10. **Return response:**
        - { success: true, submissionId, status: 'approved', approvedAt: new Date().toISOString() }

    Follow existing patterns from submit-for-review/route.ts for structure, error handling, and logging with [Approve Submission] prefix.
  </action>
