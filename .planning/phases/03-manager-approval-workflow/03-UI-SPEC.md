## Layout Patterns

**Structure:**
```
┌─────────────────────────────────────────────────────┐
│ AdminHeader: "Approvals"                            │
├─────────────────────────────────────────────────────┤
│ Filters: [Challenge ▼] [Team ▼] [Leader ▼]         │
├─────────────────────────────────────────────────────┤
│ AdminTable (ApprovalQueueTable)                     │
│ ┌───────┬──────────┬──────┬──────────┬──────────┐   │
│ │Challenge│ Student  │ Team │ Submitted │ Status   │   │
│ ├───────┼──────────┼──────┼──────────┼──────────┤   │
│ │ Mar Quiz│ John Doe │ Alpha│ 2h ago   │ Pending  │ ← Click to open detail
│ └───────┴──────────┴──────┴──────────┴──────────┘   │
└─────────────────────────────────────────────────────┘
```

**Filters (dropdowns):**
- Challenge: All challenges or specific challenge
- Team: All teams or specific team
- Leader: All leaders or specific leader
- Combined filtering: Support all combinations via URL query params

**Row click:** Opens detail view for that submission

### Approval Detail View

**Navigation:** Top bar with Previous/Next buttons

**Structure:**
```
┌──────────────────────────────────────────────────────────────┐
│ [< Previous]  [Next >]              [Approve] [Save Changes] │
├──────────────────────────────────────────────────────────────┤
│ Student: John Doe  |  Team: Alpha  |  Submitted: Mar 18, 2PM │
│ Graded by: Jane Leader  |  Submitted for review: Mar 18, 3PM │
├──────────────────────────────────────────────────────────────┤
│ Question 1: [Question text...]                              │
│ Answer: [Student answer...]                                  │
│ Auto-graded score: 10/10                                     │
│                                                              │
│ Question 2: [Essay question...]                              │
│ Answer: [Student answer...]                                  │
│ Leader grade: 8/10  Feedback: "Good answer, missing detail"   │
│ [Edit grade] → Manager can edit score and feedback           │
│                                                              │
│ ... (all questions shown)                                    │
├──────────────────────────────────────────────────────────────┤
│ Total score: 85/100                                          │
└──────────────────────────────────────────────────────────────┘
```

**Actions:**
- **Previous/Next:** Navigate between pending submissions
- **Approve:** Updates status to `approved`, notifies Leader, disables button
- **Save Changes:** Saves grade edits without approving (stays `pending_review`)

**Source:** CONTEXT.md decision: List + detail view with next/prev navigation, Manager can edit grades

### Leader Grading Interface Updates

**Existing page:** `/admin/challenges/[id]/grading`

**Additions:**
- **Per-submission button:** "Submit for Review" (when all essays graded)

## Interaction States

### Submit for Review (Leader)

**States:**
1. **Initial:** Button enabled (all essays graded)
2. **Submitting:** Loading spinner, button disabled
3. **Success:** Toast "Submission sent to Manager for review", button → "Submitted at [time]"
4. **Error:** Toast with error message, button re-enables

**Transitions:** `grading` → `pending_review`

### Review Submission (Manager)

**States:**
1. **Initial:** Detail view loads, all fields editable
2. **Editing:** Manager changes grade/feedback, "Save Changes" button enables
3. **Saving:** Loading spinner, fields disabled
4. **Saved:** Toast "Grades saved successfully", stays in `pending_review`
5. **Approving:** Loading spinner on Approve button
6. **Approved:** Toast "Grades approved", button → "Approved" (disabled), status → `approved`

**Transitions:**
- Save: `pending_review` → `pending_review` (no status change, just data update)
- Approve: `pending_review` → `approved`

### Publish to Leaderboard (Manager/Admin)

**States:**
1. **Not approved:** Button disabled, tooltip "Awaiting approval"
2. **Approved:** Button enabled (Manager/Admin only)
3. **Publishing:** Loading spinner
4. **Published:** Toast "Scores published to leaderboard", status → `published`

**Validation:**
- Check `status === 'approved'`
- Check `user.role in ['manager', 'admin']`

**Source:** CONTEXT.md decision: Manager/Admin only publish, button disabled until approved

## Accessibility Requirements

### Keyboard Navigation
- **Approval queue:** Tab through table rows, Enter to open detail view
- **Detail view:** Tab through form fields, Enter to save, Esc to close
- **Previous/Next buttons:** Arrow keys for navigation (optional enhancement)

### Screen Reader Support
- **Status badges:** ARIA labels with status text
- **Buttons:** Descriptive labels with action (e.g., "Approve grades for John Doe")
- **Filters:** Label associations with dropdowns

### Focus Management
- **Detail view open:** Focus first editable field
- **After approve:** Focus back to approval queue (or next submission)
- **Error states:** Focus first field with error

**Source:** WCAG 2.1 AA compliance (project standard)
